# 常见问题

## 基础与安装

### Q: Frpc-Hub 是什么？它和原本的 Frpc 有什么区别？

A: Frpc-Hub 是一个基于 Frpc 的图形化管理工具。它集成了 Frp 客户端的核心功能，并提供了一个现代化的 Web 界面来管理配置文件、监控状态和日志，解决了原生 Frpc 需要手动编辑配置文件和缺乏可视化监控的痛点。

### Q: Frpc-Hub 支持哪些操作系统？

A: 目前支持 Windows、macOS 和 Linux 主流发行版。

## 配置与使用

### Q: 我现有的 `frpc.toml` 配置文件可以直接导入吗？

A: 目前建议通过 Frpc-Hub 的可视化界面重新创建配置，以确保所有字段都能被正确解析和管理。

### Q: 修改配置后通过什么方式生效？

A: Frpc-Hub 支持配置热重载（Hot Reload）。在 Web 界面保存修改后，程序会自动尝试应用新的配置，通常无需重启整个服务。

### Q: 支持哪些类型的代理协议？

A: 目前完整支持 **TCP**、**UDP**、**HTTP** 和 **HTTPS** 协议。**STCP** 和 **XTCP** 等高级协议的支持正在开发中，敬请期待后续版本更新。

## 故障排查

### Q: 遇到未知错误或 Bug 该如何反馈？

A: 欢迎在 GitHub 仓库提交 Issue，请尽量提供详细的错误日志、复现步骤以及您的操作系统环境信息，以便我们快速定位问题。
